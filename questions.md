# RAG Agent Test Questions

This file contains a list of questions designed to test the "SQL Knowledge RAG Copilot" against its "hostile" database and complex business rules.

## Level 1: SQL, Casting, and Filtering (The "Hostile" Basics)
**Question:** 
"What is the total Net Realized Revenue (collected minus refunds) for all real customers across all industries?"

**What it tests:**
- **Join Path:** `organizations` -> `accounts` -> `contracts` -> `payments`.
- **Trap (Casting):** Must use `CAST(payments.contract_ref_id AS INTEGER)`.
- **Trap (Filter):** Must exclude test accounts (`organizations.is_test_account = 0`).
- **Trap (Logic):** Must calculate NRR as `status_code=1` (Settled) minus `status_code=2` (Refunded).

---

## Level 2: Semantic Mapping (The "Lexicon" Test)
**Question:**
"Show me the total revenue generated by accounts in the 'West Coast' region. Use the standard territory codes to identify the region."

**What it tests:**
- **RAG (Handbook):** Must find the "Nexus Lexicon" in `company_knowledge.md` where `'page'` = West Coast.
- **SQL:** Must query `accounts` or `sales_reps` using the correct territory code.
- **Trap:** Confirms the agent can bridge the gap between "Natural Language Region" and "Legacy Database Codes."

---

## Level 3: Hybrid Reasoning (The "Executive" Test)
**Question:**
"What is Stacy Kelly's commission for Q1 2026? Use her total revenue from the database and her specific compensation plan rules from the handbook."

**What it tests:**
- **Intent:** Should be `hybrid`.
- **SQL (Revenue):** Must calculate total payments linked to Stacy's accounts in Q1 2026.
- **SQL (Target):** Must fetch her `revenue_goal` from `rep_performance_targets` for Q1 2026.
- **RAG (Rules):** Must fetch "Plan B" rules from `sales_comp_plans.md` (15% above 100% attainment, 20% accelerator if > 120%).
- **Reasoning:** The agent must perform the math: `(Revenue / Target)` to find attainment, then apply the correct % from the MD docs.
